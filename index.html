비판적 사고 & 컴퓨팅 사고 — 데이터사이언스의 사고 기반
Critical Thinking & Computational Thinking — "무엇을 분석할지 아는 것이 어떻게 분석하는지보다 중요하다"
비판적 사고 (Critical Thinking)
정의
단순히 정보를 받아들이지 않고, 근거·논리·맥락을 따져 스스로 판단하는 능력. 데이터사이언스에서는 "이 데이터가 정말 신뢰할 수 있는가?", "모델 결과가 현실을 반영하는가?"를 끊임없이 질문하는 태도.
Paul-Elder 비판적 사고 프레임워크 (8요소)
목적 (Purpose)
분석의 목표가 무엇인가?
질문 (Question)
풀어야 할 문제는 정확히 무엇인가?
정보 (Information)
데이터는 어디서 왔고 얼마나 신뢰할 수 있나?
추론 (Inference)
결론이 데이터에서 논리적으로 도출되는가?
가정 (Assumption)
분석에 숨어있는 전제는 무엇인가?
관점 (Point of View)
다른 해석 가능성은 없는가?
개념 (Concept)
핵심 개념·용어가 명확하게 정의되었는가?
함의 (Implication)
이 결과가 현실에 어떤 영향을 미치나?
데이터사이언스에서의 가치 & 고려사항
· AI·LLM 결과를 무비판적으로 수용하지 않는 검증 습관
· 상관관계를 인과관계로 혼동하지 않는 논리적 엄밀성
· 데이터 수집·분석 과정의 편향(Bias) 인식
· 인문사회 배경 지식을 활용한 맥락 해석이 기술 역량보다 중요할 수 있음
컴퓨팅 사고 (Computational Thinking)
정의 (Jeannette Wing, 2006)
컴퓨터처럼 코딩하는 것이 아니라, 복잡한 문제를 컴퓨터가 처리할 수 있는 방식으로 재구성하는 사고 능력. 프로그래밍과 무관하게 누구나 훈련할 수 있는 범용 문제 해결 능력.
4대 핵심 요소
D
분해 (Decomposition)
큰 문제를 작고 다루기 쉬운 하위 문제로 쪼갠다. 예: "고객 이탈 예측" → 데이터 수집 / 특성 정의 / 모델 선택 / 평가 분리
P
패턴 인식 (Pattern Recognition)
유사한 문제 구조나 반복 패턴을 발견한다. 과거 분석 경험을 새로운 문제에 적용. EDA에서 패턴·이상값 탐지가 핵심.
A
추상화 (Abstraction)
핵심 요소만 남기고 불필요한 세부사항을 제거한다. 모델이 현실을 단순화하는 것 자체가 추상화. 변수 선택·특성공학의 본질.
Al
알고리즘 설계 (Algorithm Design)
문제를 해결하는 단계별 절차를 설계한다. 코드가 아닌 분석 로직의 흐름을 먼저 설계하는 습관. 가설 → 검증 → 결론.
문제 정의를 돕는 주요 방법론
디자인 씽킹
Design Thinking
공감→정의→아이디어→프로토타입→테스트. IDEO 제안. 사용자 중심으로 문제를 재정의.
5 Whys
도요타 근본원인 분석
"왜?"를 5번 반복해 표면 문제 뒤의 근본 원인을 찾는다. 간단하지만 강력한 문제 구조화 도구.
SMART 목표
목표 명확화
Specific·Measurable·Achievable·Relevant·Time-bound. 분석 목표를 측정 가능하게 구체화.
피시본 다이어그램
Fishbone / 이시카와
결과(문제)에 영향을 미치는 원인을 사람·방법·환경·측정 등 범주로 시각화. 변수 탐색에 활용.
HMW 질문법
How Might We
"어떻게 하면 우리가 ~할 수 있을까?" 형식으로 문제를 기회로 전환. 가설 생성과 분석 방향 설정에 유용.
핵심 인사이트: 비판적 사고는 "이 질문이 옳은가"를 판단하고, 컴퓨팅 사고는 "어떻게 풀 것인가"를 설계한다. 두 사고가 결합될 때 데이터사이언티스트는 AI 도구를 효과적으로 활용하면서도 결과를 주체적으로 해석할 수 있다.

1980s–1990s
데이터베이스 & 데이터마이닝
관계형 DB(SQL), 데이터웨어하우스, KDD(Knowledge Discovery in Databases — 데이터베이스 내 지식 발견) 개념 등장. 의사결정나무, 연관규칙 알고리즘 개발.
2000s
빅데이터 시대 개막
웹 2.0, Google·Amazon·Facebook 등장. Hadoop·MapReduce로 분산처리 시작. "Data Scientist" 직함 등장(DJ Patil, 2008).
2010s
머신러닝 & 클라우드 폭발
딥러닝 혁명(AlexNet 2012), AWS·Azure·GCP 확산, Spark·Kafka 등 실시간 처리. Python/R이 표준 언어로 자리잡음.
2020s
AI·LLM & AutoML 시대
GPT-3/4, BERT 등 대규모 언어모델. AutoML·MLOps 표준화. AI 윤리·편향·규제 논의 본격화.
2024–현재
Agentic AI & 바이브코딩
AI 에이전트가 분석 파이프라인 자동 수행. 자연어로 코드 생성(바이브코딩). AI Scientist 개념 등장.

STEP 3 | 데이터 전처리 & 정제
Data Preprocessing & Cleaning — "데이터의 품질이 모델의 품질을 결정한다"
주요 전처리 작업
결
결측값 처리 (Missing Values)
삭제(listwise), 평균/중앙값 대체, 회귀 대체, KNN 대체. MCAR·MAR·MNAR 유형 파악 후 방법 선택.
이
이상치 처리 (Outliers)
IQR 방법: Q1-1.5×IQR ~ Q3+1.5×IQR 범위 외 값. Z-score ±3σ. 도메인 지식으로 판단 후 제거/변환/유지.
인
인코딩 (Encoding)
범주형→수치형 변환. Label Encoding(순서 있음), One-Hot Encoding(순서 없음), Target Encoding(고카디널리티).
정
정규화/표준화 (Scaling)
Min-Max: [0,1] 범위. Z-score: 평균0·분산1. 거리 기반 알고리즘(KNN, SVM)에서 필수.
차
차원 축소 — PCA (Principal Component Analysis, 주성분분석)
고차원 데이터 압축. 다중공선성 제거. 시각화 용이.
Python 전처리 코드
import pandas as pd from sklearn.preprocessing import StandardScaler # 결측값 확인 print(df.isnull().sum()) # 수치형 결측값 → 중앙값 대체 df['age'].fillna(df['age'].median(), inplace=True) # IQR 이상치 제거 Q1 = df['income'].quantile(0.25) Q3 = df['income'].quantile(0.75) IQR = Q3 - Q1 df = df[(df['income'] >= Q1-1.5*IQR) & (df['income'] <= Q3+1.5*IQR)] # One-Hot 인코딩 df = pd.get_dummies(df, columns=['gender']) # 표준화 (StandardScaler) scaler = StandardScaler() df[['age','income']] = scaler.fit_transform( df[['age','income']])

STEP 4 | 탐색적 데이터 분석 (EDA)
EDA — Exploratory Data Analysis | 데이터를 이해하는 탐정의 시간
EDA의 목적과 핵심 질문
01
분포 파악 (Distribution)
각 변수의 분포 형태, 중심경향·산포도. 정규분포인가? 왜도·첨도는?
02
관계 탐색 (Relationship)
변수 간 상관관계. 산점도·상관행렬·히트맵으로 시각화.
03
그룹 비교 (Comparison)
범주별 차이 존재하는가? 박스플롯, 바이올린플롯으로 비교.
04
패턴·이상 발견 (Pattern)
시계열 추세, 주기성, 군집, 예상치 못한 이상값 발견.
기술통계 핵심 지표
중심경향 (Center)
평균(Mean): 극단값에 민감
중앙값(Median): 이상치에 강건
최빈값(Mode): 범주형에 유용
산포도 (Spread)
분산(Variance): σ²
표준편차(SD, Standard Deviation): σ
IQR: Q3-Q1
형태 (Shape)
왜도(Skewness): 대칭 여부
첨도(Kurtosis): 꼬리 두께
정규성 검정
관계 (Relation)
피어슨 상관: 선형 관계
스피어만 상관: 순위 관계
크래머 V: 범주형
시각화 차트 선택 가이드
분포 시각화 (Distribution)
히스토그램 (Histogram): 연속형 분포
박스플롯 (Box Plot): 사분위·이상치
바이올린플롯 (Violin Plot): 분포 형태
KDE 밀도곡선 (Kernel Density Estimate)
관계 시각화 (Relationship)
산점도 (Scatter Plot): 두 변수 관계
버블차트 (Bubble Chart): 3변수
페어플롯 (Pair Plot): 다변수 행렬
히트맵 (Heatmap): 상관행렬
시계열 시각화 (Time Series)
선 그래프 (Line Chart): 추세
구간차트 (Area Chart): 불확실성
계절분해 (Seasonal Decomposition): STL
이동평균선 (Moving Average)
STL = Seasonal and Trend decomposition using Loess
Loess(국소 가중 회귀)를 활용해 시계열을 계절성·추세·잔차로 분해하는 기법
구성 시각화 (Composition)
막대차트 (Bar Chart): 범주 비교
파이차트 (Pie Chart): 비율 (5개 이내)

핵심 통계 이론 & 주요 개념
Statistical Foundations — 데이터사이언스의 수학적 기반
추론통계 (Inferential Statistics)
가설검정 (Hypothesis Testing) 초급
"내가 관찰한 차이나 효과가 진짜인가, 아니면 우연인가?"를 수학적으로 판단하는 방법.
예: "남녀 임금 차이가 실제로 존재하는가, 우연한 샘플 차이인가?"
H₀(귀무가설): 차이 없음 vs H₁(대립가설): 차이 있음
p-value < α(보통 0.05) → 귀무가설 기각 → "차이가 있다"
언제 쓰나: 두 그룹 비교(t-검정), 3개 이상 그룹(ANOVA), 범주형 변수 관계(카이제곱 χ² 검정)
주의: p-value는 효과 크기가 아님. 표본이 크면 사소한 차이도 유의해질 수 있음
중심극한정리 (CLT — Central Limit Theorem) 초급
모집단이 어떤 분포든, 샘플을 충분히 뽑아 평균을 내면 정규분포에 가까워진다는 원리.
예: 주사위는 균등분포지만 100번 던진 평균들은 종 모양(정규분포)으로 모임.
표본평균 X̄ ~ N(μ, σ²/n) (n이 클수록 정규분포에 근사)
의미: 통계적 추론이 가능한 근거. 현실에서 정규분포 가정을 쓸 수 있는 이론적 토대.
베이즈 정리 (Bayes' Theorem) 중급
새로운 증거(데이터)를 보고 기존의 믿음(확률)을 업데이트하는 방법.
예: 코로나 검사 양성이어도 거짓양성률을 고려해 실제 감염 확률을 재계산.
P(A|B) = P(B|A) × P(A) / P(B)
언제 쓰나: 스팸 필터(나이브 베이즈), A/B 테스트 해석, 의료 진단, 추천 시스템
엔트로피 (Entropy) — 정보 이론 중급
불확실성의 양을 수학적으로 측정. "얼마나 많은 정보가 담겼는가?"
동전 던지기(50:50)는 엔트로피 최대, 확실한 결과는 엔트로피 0.
H(X) = -Σ p(x) × log₂p(x)
언제 쓰나: 의사결정나무 분기 기준(정보이득), 딥러닝 손실함수(교차엔트로피 Cross-Entropy)
회귀분석 & 핵심 개념
선형회귀 (Linear Regression) 초급
두 변수 사이 선형 관계를 직선으로 표현.
예: "학습 시간이 1시간 늘면 시험 점수가 평균 몇 점 오르나?"
ŷ = β₀ + β₁x₁ + β₂x₂ + ε
평가 지표:
R² (결정계수): 모델이 데이터 분산을 얼마나 설명하는가 (0~1)
RMSE: 예측 오차의 평균적 크기 (단위: 원본 변수와 동일)
p-value: 각 독립변수가 통계적으로 유의한가 (<0.05 이면 유의)
편향-분산 트레이드오프 (Bias-Variance Tradeoff) 중급
모델이 너무 단순하면 → 편향(Bias) 큼 → 중요한 패턴을 놓침 → 과소적합 (Underfitting)
모델이 너무 복잡하면 → 분산(Variance) 큼 → 훈련 데이터 잡음까지 외움 → 과적합 (Overfitting)
총 오류 = 편향² + 분산 + 줄일 수 없는 오류(Irreducible Error)
해결책: 정규화(Regularization) — L1(Lasso), L2(Ridge)로 복잡도 제어
언제 쓰나: 훈련 오류는 낮은데 테스트 오류가 높을 때 → 과적합 의심
상관관계 ≠ 인과관계 초급
두 변수가 함께 움직인다고 해서 한쪽이 원인인 것은 아님.
예: "아이스크림 판매량과 익사 사고 동시 증가" → 원인은 '더운 날씨'(교란변수, Confounding Variable)
인과관계 파악법 (고급):
무작위 대조실험 (RCT) 중급 / 이중차분법 DID 고급 / 도구변수법 IV 고급

Machine Learning Algorithms — 언제 무엇을 쓸까
머신러닝 분류 체계 & 주요 알고리즘
지도학습 (Supervised Learning)
정답(레이블)이 있는 데이터로 학습. 입력→출력 매핑.
회귀 분류
로지스틱 회귀초급
"합격/불합격", "이탈/유지"처럼 두 가지 중 하나를 예측하는 분류 모델.
쓸 때: 이진분류, 계수로 영향력 해석 필요 시
장점: 빠름, 결과 해석 쉬움
단점: 비선형 패턴 취약
랜덤포레스트초급
여러 의사결정나무가 각자 예측 → 다수결로 최종 결정. 한 나무의 실수를 다른 나무들이 보완.
쓸 때: 다양한 문제의 기본 베이스라인, 변수 중요도 파악
장점: 높은 성능, 과적합 강건
단점: 해석 어려움

이전 나무가 틀린 부분을 다음 나무가 집중 보완하며 순차적으로 성능 향상(부스팅).
쓸 때: Kaggle 대회, 정형 데이터 예측 경진
단점: 하이퍼파라미터 튜닝 필요
비지도학습 (Unsupervised Learning)
정답 없이 데이터 스스로의 구조·패턴 발견. "이 데이터들은 어떻게 묶이나?"
군집화 차원축소
K-Means 군집화초급
비슷한 데이터끼리 K개 그룹으로 묶기. 중심점을 반복 이동해 최적 그룹 탐색.
쓸 때: 고객 세분화, 뉴스 군집, 설문 응답 유형화
주의: K를 미리 정해야 함, 이상치에 민감
PCA (Principal Component Analysis, 주성분분석)중급
수십 개 변수를 정보 손실 최소화하며 2~3개로 압축. 설문 50문항 → 핵심 요인 3개 추출.
쓸 때: 고차원 데이터 시각화, 다중공선성 제거
LDA 토픽 모델링 (Latent Dirichlet Allocation)중급
문서 집합에서 숨겨진 주제 자동 발견. "뉴스 1000건 → 정치/경제/스포츠 자동 분류"
쓸 때: 정책 문서 분석, 여론조사 텍스트, 역사 자료
기타 핵심 알고리즘
문제 유형과 데이터 특성에 따라 선택.
SVM (Support Vector Machine, 서포트 벡터 머신)중급
두 그룹을 가장 넓은 여백(Margin)으로 구분하는 경계선 찾기. 텍스트 분류에 강력.
쓸 때: 텍스트·이미지 분류, 소규모 고차원 데이터
KNN (K-Nearest Neighbors, K-최근접 이웃)초급
새 데이터와 가장 가까운 K개 이웃의 다수결로 분류. "비슷한 사람은 비슷하게 행동한다"
쓸 때: 개념 이해용, 추천 시스템 기초
단점: 대용량에서 매우 느림
앙상블 (Ensemble) — 여러 모델 조합중급
여러 모델의 예측을 결합해 단일 모델보다 더 나은 성능 달성.
배깅(병렬 결합) / 부스팅(순차 보완) / 스태킹(층위 조합)
대표: 랜덤포레스트(배깅), XGBoost(부스팅)

STEP 6 | 모델 검증, 평가 & 해석
Model Validation, Evaluation & Interpretation
분류 모델 평가 지표
혼동행렬 (Confusion Matrix) 초급
모델의 예측 결과를 실제 정답과 대조한 표. 어디서 어떻게 틀렸는지 한눈에 파악.
예측: Positive
예측: Negative
실제: Positive
TP (정답)
FN (놓침)
실제: Negative
FP (오탐)
TN (정답)
분류 평가 4가지 초급
정확도 (Accuracy) = (TP+TN) / 전체 — "전체 중 맞힌 비율"
불균형 데이터에서는 무의미할 수 있음 (95% 정상 데이터를 다 정상이라 해도 95%)
정밀도 (Precision) = TP / (TP+FP) — "양성 예측 중 실제 양성 비율"
오탐 비용이 클 때 중요 (예: 무고한 사람을 범인으로 지목하면 안 될 때)
재현율 (Recall/Sensitivity) = TP / (TP+FN) — "실제 양성 중 발견한 비율"
놓침 비용이 클 때 중요 (예: 암 환자를 정상으로 오진하면 안 될 때)
F1 Score = 2×(Precision×Recall)/(Precision+Recall) — 두 지표의 조화평균
ROC-AUC 중급
ROC = Receiver Operating Characteristic (수신자 조작 특성 곡선) | AUC = Area Under the Curve (곡선 아래 면적)
임계값을 바꿔가며 True Positive율 vs False Positive율 트레이드오프 시각화.
AUC = 1: 완벽 모델 / AUC = 0.5: 랜덤 수준 / 불균형 데이터에 특히 유용
회귀 모델 평가 지표
MAE 초급
Mean Absolute Error — 평균 절대 오차
1/n Σ|y - ŷ|
오차 절댓값의 평균. 이상치 영향 적음. 직관적 해석 용이.
RMSE 초급
Root Mean Squared Error — 평균 제곱근 오차
√(1/n Σ(y - ŷ)²)
큰 오차에 더 민감. 단위가 원본과 동일해 해석 용이.
R² (결정계수) 초급
Coefficient of Determination — 결정계수
1 - SS_res / SS_tot
0~1, 모델이 전체 분산을 얼마나 설명하는가.
MAPE 초급
Mean Absolute Percentage Error — 평균 절대 백분율 오차
1/n Σ|y - ŷ| / y × 100%
퍼센트 단위 오차. 서로 다른 모델·데이터 비교에 유용.
교차검증 & 모델 해석
K-Fold 교차검증 (Cross-Validation) 초급
데이터를 K개로 나눠 번갈아 검증. 전체 데이터를 한 번씩 테스트에 사용해 성능을 더 신뢰있게 추정. K=5 또는 10 권장.

게임이론 기반. 각 변수가 예측에 기여한 정도를 수치로 표현.

데이터사이언스 윤리 & 주요 오류 유형
Data Science Ethics & Common Pitfalls — "데이터도 틀릴 수 있다"
주요 통계·분석 오류 유형
알고리즘 편향 (Algorithmic Bias) 초급
훈련 데이터에 존재하는 사회적 편견이 모델에 그대로 반영되는 현상.
예: 특정 인종 얼굴인식 오류율 높음, 채용 AI가 여성 지원자 불이익.
해결책: 공정성(Fairness) 지표 측정, 다양한 데이터 수집, 알고리즘 감사(Audit)
p-해킹 / HARKing 중급
p-해킹: p-value가 0.05 이하가 나올 때까지 분석 방법·변수를 바꾸는 행위.
HARKing: Hypothesizing After Results Known — 결과를 보고 나서 가설을 만드는 행위.
해결책: 사전 등록(Pre-registration), 복제 연구, 다중검정 보정(Bonferroni)
심슨의 역설 (Simpson's Paradox) 중급
전체 데이터 분석 결과와 하위 그룹별 분석 결과가 반대로 나타나는 현상.
예: 전체적으로는 치료법 A가 좋아 보이지만, 남녀 각각 분석하면 B가 더 좋음.
해결책: 그룹별 층화 분석 필수. 교란변수 통제.
확증 편향 (Confirmation Bias) 초급
기존 믿음을 확인하는 데이터만 선택적으로 수집·해석하는 경향.
예: "이미 알고 있는 결론"을 뒷받침하는 분석만 보고하는 행위.
해결책: 반증 사례 적극 탐색, 외부 검토자 활용
AI 거버넌스 & 책임있는 AI

잘린 축, 선택적 기간, 3D 효과 등.

EU AI법(AI Act): 고위험 AI는 의사결정 근거 설명 의무화.
블랙박스 모델(딥러닝)의 결정 과정을 설명할 수 있어야 함.
도구: SHAP, LIME, Grad-CAM (이미지 설명)

AI 에이전트 & 바이브코딩이 바꾸는 데이터사이언스
Agentic AI & Vibe Coding — 2024년 이후 가장 큰 패러다임 전환
AI 에이전트란?
AI 에이전트 (Agentic AI)
사람의 단계별 지시 없이 스스로 계획을 세우고, 도구를 사용하고, 결과를 검증하며 목표를 달성하는 AI.
예: "이 CSV를 분석해서 인사이트를 담은 보고서를 만들어줘" → AI가 코드 작성·실행·시각화·보고서 작성을 자동으로 수행.
핵심 구성요소:
계획(Planning) → 도구 사용(Tool Use) → 메모리(Memory) → 자기 검증(Self-verification)
바이브코딩 (Vibe Coding)
자연어로 원하는 것을 설명하면 AI가 코드를 작성해주는 방식으로 프로그래밍하는 새로운 패러다임.
코드를 직접 작성하는 대신 AI와 대화하며 의도(Vibe)를 전달.
도구: GitHub Copilot, Claude, ChatGPT Code Interpreter, Cursor, Replit AI
의미: 프로그래밍 문법 지식이 없어도 데이터 분석 자동화 가능 → 데이터사이언스 민주화
Text-to-SQL / NL2Data
자연어로 데이터베이스를 조회하는 기술.
"지난 달 가장 많이 팔린 상품 10개는?" → SQL 자동 생성 → 결과 반환.
의미: 비전문가도 데이터에 직접 접근 가능. 데이터 민주화의 실현.
이 시대 데이터사이언티스트의 역할
문제 정의자 & 도메인 전문가
AI는 코드를 쓸 수 있지만 "무엇을 물어야 하는가"는 사람이 정해야 함.
인문사회 도메인 지식으로 올바른 질문을 설계하고 분석 방향 결정.
AI 결과 검증자 & 품질 관리자
AI가 생성한 코드·분석·시각화의 오류를 발견하고 수정.
"AI는 자신있게 틀릴 수 있다(Hallucination)" — 통계 지식으로 결과의 타당성 검증.
프롬프트 엔지니어 & AI 오케스트레이터
AI 에이전트에게 효과적으로 지시하는 프롬프트 설계.
여러 AI 에이전트와 도구를 조합한 분석 파이프라인 설계.
윤리·편향 감시자
AI가 만든 분석이 특정 집단에 불이익을 주지 않는지 감시.
알고리즘 공정성 검토, 개인정보 보호 가드레일 설계.
커뮤니케이터 & 스토리텔러
AI가 만든 분석 결과를 의사결정자·대중이 이해할 수 있게 설명.
인문사회 맥락에서의 해석과 정책적 함의 제공.
핵심 메시지
"코딩을 AI가 대신하는 시대에, 데이터사이언티스트의 가치는
무엇을 분석할지 알고, 결과가 맞는지 판단하며, 의미를 해석하는 능력에 있다."
인문사회계열의 비판적 사고, 사회 이해, 맥락 해석 능력이 핵심 경쟁력.

RAG — Retrieval-Augmented Generation (검색 증강 생성)
LLM의 한계를 넘는 핵심 아키텍처 — "모르면 찾아서 답한다"
개념 & 작동 구조
RAG란 무엇인가?
LLM(Large Language Model)은 학습된 데이터까지만 알고, 최신 정보·내부 문서는 모른다. RAG는 질문을 받으면 ① 외부 데이터베이스에서 관련 문서를 검색(Retrieve)하고 ② 그 내용을 맥락으로 LLM에 전달해 ③ 사실 기반 답변을 생성(Generate)하는 방식이다.
한 줄 요약: 인터넷 검색 + AI 답변 생성을 결합한 구조 — Hallucination(환각) 문제를 줄이는 핵심 기법

핵심 구성요소 & 현재 기술동향
벡터 데이터베이스 (Vector DB)
텍스트를 고차원 벡터로 변환해 의미 기반 유사도 검색. 주요 도구: Pinecone, ChromaDB, Weaviate, FAISS. "사과"와 "과일"이 유사하다는 의미를 숫자로 표현.
임베딩 모델 (Embedding)
텍스트 → 수치 벡터 변환. OpenAI text-embedding, SBERT, BGE 등. 한국어는 KR-SBERT, Upstage Solar Embeddings 활용.

GraphRAG: 지식그래프 + RAG (MS 오픈소스)

주요 NLP 기법
TF-IDF 초급
TF = Term Frequency (단어 빈도), IDF = Inverse Document Frequency (역문서빈도). 많이 나오면서도 해당 문서에서만 특이하게 나오는 단어가 중요하다는 원리. 키워드 추출, 문서 유사도 계산.
TF-IDF = Term Frequency-Inverse Document Frequency
LDA 토픽 모델링 중급
문서 집합에서 숨겨진 토픽(주제)을 자동으로 발견. 수백 개의 뉴스 기사에서 자동으로 "경제", "정치", "문화" 토픽을 추출. 인문사회 연구에 특히 유용.
LDA = Latent Dirichlet Allocation (잠재 디리클레 할당)
감성분석 (Sentiment Analysis) 초급
텍스트에서 긍정/부정/중립 감성을 자동 분류. 리뷰 분석, 여론조사, SNS 모니터링. 사전 기반(KNU 한국어 감성사전) vs 딥러닝 기반(KoBERT) 방식.
BERT & 트랜스포머 고급
양방향 문맥 이해로 문장 전체 의미를 파악. 한국어 특화: KoBERT, KoELECTRA. 한국어 BERT 기반 감성분석은 Hugging Face에서 바로 사용 가능.
BERT = Bidirectional Encoder Representations from Transformers

공간데이터 분석 (Spatial Data Analysis)
GIS & Geospatial Analytics — "위치 정보가 담긴 데이터로 세상을 읽는 방법"
공간데이터란?
정의
지리적 위치(위도·경도)나 공간 관계를 포함하는 데이터. 지도, GPS 트랙, 주소, 행정구역 경계 등. GIS(Geographic Information System) 도구로 분석하며, "어디서 무슨 일이 일어나는가?"에 답한다.
GIS = Geographic Information System (지리정보시스템)
공간 분석 주요 기법
지도 시각화 (Choropleth Map)
행정구역별 데이터를 색상 강도로 표현. 시군구 인구, 소득, 범죄률 시각화.
공간 클러스터링
지리적으로 밀집된 패턴 발견. 핵스팟 분석(Hot Spot), DBSCAN 활용.
네트워크 분석
도로망, 교통 흐름, 사회적 연결망 분석. 최단경로, 접근성 계산.
공간 자기상관 (Moran's I)
지리적으로 인접한 지역끼리 유사한 값을 가지는지 측정. 공간 의존성 검증.
Python 공간분석 도구
Folium — 인터랙티브 지도
Leaflet.js 기반 웹 지도 생성. 마커, 히트맵, 클로로플레스 맵. 결과를 HTML로 바로 저장·공유 가능.
GeoPandas — 공간 데이터프레임
Shapefile, GeoJSON 읽기·분석·시각화. 공간 조인, 버퍼 분석, 면적 계산 등 GIS 연산을 Pandas처럼 수행.
공공 공간데이터 소스
국토정보플랫폼 (V-World)
공공데이터포털 (행정구역 GeoJSON)
통계청 통계지리정보서비스 (SGIS)

설문조사 데이터
수집
리커트 쳛도, 선택형, 서술형 혼합 설계. IRB 승인 필수.

리뷰·SNS 텍스트 데이터
수집
크롤링(BeautifulSoup), API(Twitter/X, Naver), 공개 데이터셋
전처리
HTML 태그 제거, 이모지 처리, 중복 제거, 형태소 분석(KoNLPy)
분석
TF-IDF 키워드 → 감성분석 → LDA 토픽 모델링 → 시계열 트렌드
시각화
워드클라우드, 감성 시계열, 토픽 비율 스택 차트
도구: KoNLPy / gensim / transformers (KoBERT)
공간·위치 데이터
수집
공공데이터포털 GeoJSON, 통계청 SGIS, GPS 로그, OpenStreetMap
전처리
좌표계 통일(EPSG:4326), 공간 조인, 경계 정리, 이상 좌표 제거
분석
클로로플레스 매핑 → 공간 클러스터링 → 접근성 분석 → Moran's I
시각화
Folium 인터랙티브 지도, Plotly 3D 지도, Kepler.gl
도구: GeoPandas / Folium / PySAL / QGIS

공간데이터 선택 시: 지역 격차, 도시 분석, 환경·사회 불평등의 공간적 분포 파악에 적합.
